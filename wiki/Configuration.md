## Source of truth

- **`backend/app/config.py`** — the `Settings` Pydantic model. Every variable Nyx reads is declared here with its default. If the README disagrees with this file, the file wins.
- **`.env.example`** — the starter template `setup.sh` copies to `.env` on first run.
- **`docker-compose.yml`** / **`docker-compose.postgres.yml`** — the variables actually injected into the backend container.

## Quick-reference: the 15 most important variables

| Variable | Required for | Default | Notes |
|---|---|---|---|
| `GITHUB_TOKEN` | All GitHub features | — | PAT with `repo`, `admin:repo_hook`, `security_events` scopes |
| `ANTHROPIC_API_KEY` | AI fixes, Auto PR Mode | — | Get from [console.anthropic.com](https://console.anthropic.com/settings/keys) |
| `GITHUB_WEBHOOK_ENDPOINT` | Webhook delivery | — | **Full path required:** `.../api/v1/webhooks/github`. After changing, run `docker compose up -d`. |
| `NYX_SECRET_KEY` | Encryption, audit HMAC | _(generated)_ | Auto-generated by `setup.sh`. Changing it breaks the audit HMAC chain. |
| `NYX_API_KEY` | Dashboard login | _(generated)_ | Auto-generated bootstrap key is admin-scoped. Create scanner-scoped keys for CI. |
| `NYX_WEBHOOK_SECRET` | Global webhook pre-check | _(empty)_ | **Leave empty** in standard setups. See [`.env.example`](../.env.example) for when to set it. |
| `DATABASE_URL` | Everything | `sqlite+aiosqlite:///./data/nyx.db` | Switch to PostgreSQL for production. |
| `AUTO_PR_MODE_ENABLED` | Auto PR Mode | `false` | Master switch — per-repo opt-in also required via UI toggle. Also gates the daily budget-reset loop; must be `true` or per-repo budgets never reset. |
| `AUTO_PR_MAX_CONCURRENT` | Auto PR concurrency | `3` | Global semaphore: max parallel auto-PR pipelines across all repos. |
| `AUTO_PR_CHECK_TIMEOUT` | CI check wait | `600` | Seconds to wait for a GitHub Actions check run before timing out. |
| `AUTO_PR_BLOCK_ON_TIMEOUT` | CI timeout behavior | `false` | When `true`, a check-run timeout blocks the commit; when `false`, the PR is opened regardless. |
| `AUTO_PR_FIX_MODEL` | Fix generation model | `claude-sonnet-4-6` | Model used by the Auto PR fix-generation pass. |
| `AUTO_PR_AUDIT_MODEL` | Security audit model | `claude-haiku-4-5` | Cheaper model for the independent security-audit pass and advisory guidance. |
| `ANTHROPIC_MODEL` | AI fixes, Auto PR | `claude-sonnet-4-6` | |
| `LOG_LEVEL` | Logging | `INFO` | |
| `ENVIRONMENT` | Security hardening | `development` | Set to `production` to disable API docs, enforce HTTPS cookies. |
| `CORS_ORIGINS_STR` | Frontend access | `http://localhost:5173,http://localhost:3000` | Comma-separated allowed origins. |
| `CODE_SCANNING_SYNC_ENABLED` | GitHub Code Scanning import | `false` | Requires `security_events` scope on `GITHUB_TOKEN`. |
| `REQUIRE_SUBMISSION_HMAC` | CI scan integrity | `false` | Set `true` in production to reject unsigned scan submissions. |
| `SLA_CRITICAL_DAYS` | SLA tracking | `7` | Days before a CRITICAL finding is flagged as SLA-breached. |

> For the full list of every variable with types, defaults, and validation, see **[`backend/app/config.py`](../backend/app/config.py)**.

## Adding a new variable

1. Declare it on `Settings` in `backend/app/config.py` with a sensible default.
2. Add it to `.env.example` with an explanatory comment.
3. Add a row to the relevant `<details>` block in the README's Configuration Reference section.
4. If it changes how a user *sets up* Nyx, also mention it in [Installation](Installation.md).
